# 知識アセスメントと知識提供 リファレンス

## 知識レベルの判定基準

### 判定方法

ユーザーの発言から以下の信号を読み取る：

**専門用語の使用**
- 正確な専門語を使っている → 基礎知識あり
- 一般語・比喩で表現している → 基礎知識の確認が必要
- 誤用・混同した専門語を使っている → 修正が必要

**問いの具体性**
- 「建築基準法の接道義務の例外規定について」→ 具体的・専門的
- 「なぜ古い建物は建て替えが難しいの？」→ 概念的・一般的

**背景説明の有無**
- 自ら文脈・背景を説明できている → 相応の知識あり
- 背景説明なしに結論を求めている → 前提の共有が必要

### 知識レベルの3分類

**Level A: 専門家・実務者レベル**
- 建築士、研究者、都市計画担当者など
- 対応: 前提確認を省略し、議論に直入する
- 注意: 専門分野が異なる場合があるため、領域はじめに確認する

**Level B: 関心ある一般人・他分野の専門家**
- 建築に関心があるが、専門的な訓練はない
- 対応: 核心的な概念・制度を簡潔に説明してから議論する
- 注意: 専門語の過度な使用を避け、例を多用する

**Level C: 初学者・一般市民**
- 建築の専門知識が少ない
- 対応: 議論の前提となる基礎概念から丁寧に説明する
- 注意: 議論の主題そのものを理解するところから始める

## 領域別の背景知識チートシート

### 建築法規・制度

**頻出の知識ギャップ:**
- 建築確認申請と完了検査の違い
- 用途地域と建蔽率・容積率の関係
- 既存不適格建築物の概念（法改正前に合法だったが現在は不適格）
- 接道義務と再建築不可物件

**提供すべき最低限の知識:**
```
建築基準法は「最低限の安全基準」を定める法律。
建物を建てるには「建築確認申請」が必要で、設計段階で法適合を確認する。
「用途地域」が建物の用途・規模を制限し、地域によって建てられるものが異なる。
```

### 構造・耐震

**頻出の知識ギャップ:**
- 新耐震基準（1981年）と旧耐震基準の違い
- 許容応力度設計と限界耐力計算の違い
- 免震・制振・耐震の違い

**提供すべき最低限の知識:**
```
日本の耐震基準は1981年に大きく改正（新耐震基準）。
「耐震」は揺れに耐える設計、「免震」は揺れを建物に伝えない設計、
「制振」は揺れのエネルギーを吸収する設計。
```

### 都市計画・まちづくり

**頻出の知識ギャップ:**
- 都市計画法と建築基準法の役割分担
- 地区計画と用途地域の関係
- 空き家対策特別措置法の概要
- 再開発事業の種類（市街地再開発、土地区画整理）

**提供すべき最低限の知識:**
```
「都市計画法」が街全体の土地利用を決め、
「建築基準法」が個々の建物の基準を定める。
空き家対策は2015年の「空き家対策特別措置法」で自治体の権限が強化された。
```

### 伝統構法・木造建築

**頻出の知識ギャップ:**
- 在来工法と伝統構法の違い
- 木造軸組と枠組壁工法（ツーバイフォー）の違い
- 伝統構法の建築確認申請の困難さ（法的根拠の問題）

**提供すべき最低限の知識:**
```
「伝統構法」は継手・仕口などの木組みで力を伝達する日本の伝統的な木造技術。
現行の建築基準法は軸組計算を前提とするため、伝統構法の法的な適合確認が難しい。
```

### 省エネ・環境

**頻出の知識ギャップ:**
- 省エネ法と建築省エネ法の関係
- ZEH（ゼロエネルギーハウス）の定義
- CASBEE、BELSなどの建築環境性能評価

**提供すべき最低限の知識:**
```
建築物省エネ法（2016年）により、新築建物への省エネ基準適合が義務化された（段階的拡大）。
ZEHは年間の一次エネルギー消費量をゼロ以下にすることを目標とした住宅。
```

## 知識提供の実施手順

### 知識提供前のチェック

1. ユーザーが既に知っている可能性を確認する
   - 「〇〇についてはご存知ですか？」と直接確認
   - 知っている場合はスキップし、知らない場合のみ提供

2. 議論への必要性を確認する
   - その知識がないと、後続の議論がどこで詰まるかを予測する
   - 直接関係しない周辺知識は提供しない

### 知識提供の原則

**過不足をなくす**
- 必要な概念だけを説明する（関連する全知識を提供しない）
- 1回の説明で3概念以上を詰め込まない

**具体例を使う**
- 抽象的な説明には必ず具体例を添える
- 日本の身近な事例（東京、大阪、地方都市）を使う

**確認して進む**
- 知識提供後「ご理解いただけましたか？この前提で議論を進めてよいですか？」と確認
- ユーザーから追加質問が出た場合は答えてから次へ進む

### 知識提供の形式

```markdown
**[概念名]について補足します**

[2〜4文の簡潔な説明]

例えば: [具体的な事例で説明]

この知識を踏まえて、改めて問いを整理すると: [議論への接続]
```

## 誤解の修正手順

ユーザーが事実と異なる前提を持っている場合：

1. 直接否定せず、「別の見方もあります」と前置きする
2. 正確な情報と出典を提示する
3. なぜその誤解が生じやすいかを説明する（ユーザーを責めない）
4. 修正された前提で議論を再開する

例:
```
「新耐震基準以降の建物は安全」という理解はよく見られますが、
実際には「新耐震基準を満たしている = 大地震で倒壊しない水準は確保されているが、
損傷がないわけではない」という意味です。
2016年熊本地震では新耐震基準の建物にも損傷が見られました。
この点を踏まえると、「耐震性」の議論がより正確になります。
```
