# 建築調査ソース戦略 詳細リファレンス

## 日本語主要ソース一覧

### 法令・行政

| ソース | URL | 用途 | アクセス方法 |
|---|---|---|---|
| e-Gov法令検索 | https://laws.e-gov.go.jp | 建築基準法、都市計画法、消防法など法令全文 | WebSearch → WebFetch |
| 国土交通省 | https://www.mlit.go.jp | 建築・住宅政策、告示、通達 | WebSearch `site:mlit.go.jp` |
| 建築基準法テキスト | laws.e-gov.go.jp/law/325AC0000000201 | 建築基準法全文 | WebFetch直接 |
| 各都道府県建築指導課 | 各県サイト | 地域固有の条例・取扱い | WebSearch `[都道府県名] 建築指導課` |

**建築基準法調査の手順:**
1. `建築基準法 第〇条 e-gov` でWebSearch
2. e-Gov法令検索のURLをWebFetchで取得
3. 関連する施行令・省令も確認（条文間の参照が多い）
4. 改正年月日を必ず確認（最終改正日が重要）

### 学術データベース

| ソース | 特徴 | アクセス方法 |
|---|---|---|
| J-STAGE | 国内学術論文、無料アクセス多数 | WebSearch `[テーマ] site:jstage.jst.go.jp` |
| CiNii Research | 論文・図書・研究データ統合検索 | WebSearch `[テーマ] site:cir.nii.ac.jp` |
| 建築学会電子図書館 | 日本建築学会論文集 | 会員制（概要のみ無料の場合あり） |
| Google Scholar | 広域学術検索 | WebSearch `[テーマ] 建築 filetype:pdf` |

**J-STAGE調査の手順:**
1. WebSearch: `[テーマ] 建築学会 J-STAGE`
2. 検索結果からJ-STAGEのURLを特定
3. WebFetchで論文要旨・PDF URLを取得
4. PDF URLが取得できればWebFetchで本文取得を試みる
5. 有料の場合は要旨と引用情報のみ記録する

**CiNii調査の手順:**
1. WebSearch: `[テーマ] CiNii`
2. CiNiiの検索結果ページをWebFetchで取得
3. 論文リストを確認し、オープンアクセスのものを選ぶ
4. DOIまたはPDF URLからWebFetchで取得

## クエリ設計の詳細パターン

### 精密なクエリ設計

**狭く深く調べる場合:**
```
"建築基準法" "接道義務" "例外" 2023
"木造" "伝統構法" "建築確認" "現行制度"
```

**広く浅く調べる場合（全体把握）:**
```
建築 空き家 対策 最新
日本 建築 省エネ 動向
```

**事実確認のクエリ:**
```
[数値] [根拠を示す語: 統計 調査 報告書]
例: "空き家率" "13.6%" 統計
```

**歴史・変遷を調べる場合:**
```
建築基準法 改正 歴史 変遷
木造建築 耐震 歴史 [年代]
```

### クエリの調整方法

最初のクエリで期待する結果が得られない場合：
1. 専門語を一般語に変える（または逆）
2. 年代を絞る / 外す
3. サイト限定を追加する（`site:mlit.go.jp`）
4. 英語クエリに切り替える

## 信頼性評価の詳細基準

### 一次情報と二次情報の見分け方

**一次情報（直接的根拠）:**
- 法令原文、政府統計、学術論文、公式報告書
- 設計者・建築主による作品説明

**二次情報（解釈・転載）:**
- 新聞・雑誌記事（一次情報を引用）
- ウェブコラム、ブログ
- Wikipedia

**原則:** 二次情報を見つけたら、引用元の一次情報を辿る。

### 情報の新鮮さの確認

- 法令: 最終改正日を確認（e-Govに記載）
- 統計: 調査年次を確認（古いデータを現在の議論に使わない）
- 事例: 竣工年・掲載年を確認
- 研究: 発表年と現在の状況の差異を考慮

### ドメイン別の信頼性スコアリング

```
gov.go.jp / go.jp  → ★★★★★ (政府機関)
jstage.jst.go.jp   → ★★★★★ (査読済み学術論文)
nii.ac.jp          → ★★★★☆ (学術データベース)
ac.jp / edu        → ★★★★☆ (大学・研究機関)
or.jp              → ★★★☆☆ (公益法人・業界団体)
co.jp / com        → ★★☆☆☆ (企業 — 自社都合の情報に注意)
個人ブログ等       → ★☆☆☆☆ (要一次情報確認)
```

## 調査が難しいケースの対処法

### 有料論文の場合

1. 要旨（アブストラクト）のみで議論に使えるか判断する
2. 著者機関の大学リポジトリでオープンアクセス版を探す
   - WebSearch: `[論文タイトル] 大学名 repository OR 機関リポジトリ`
3. ResearchGateで著者がアップロードしていないか確認する
4. 取得できない場合は「要旨レベルでは〇〇と考えられる。本文確認が必要」と明記する

### 情報が見つからない場合

1. クエリを変えて3回まで試す
2. 英語で探す
3. 専門家の著書・レポートを参照する
4. 「調査範囲内では確認できなかった」と明記し、追加調査の提案をする

### 矛盾する情報が見つかった場合

1. 両方の情報と出典を記録する
2. 発表年・調査方法・対象スコープの違いを確認する
3. 信頼性の高いソース（政府・査読論文）を優先する
4. 「ソースによって異なる。信頼性の観点では〇〇が優先される」と結論付ける
